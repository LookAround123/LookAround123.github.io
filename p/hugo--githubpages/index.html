在 Hugo 中，Front Matter 是 Markdown 文件的元数据部分，用于定义页面的配置和属性。除了你已经看到的属性（如 `author`、`title`、`date` 等），Hugo 支持许多其他 Front Matter 属性，这些属性可以根据你的需求和主题（theme）进行定制。以下是一些常见的 Front Matter 属性及其用途：

### **1. 基础属性**
这些是 Hugo 中常用的 Front Matter 属性，适用于大多数页面类型：

- **`title`**：页面标题。
- **`date`**：页面的发布日期，格式为 `YYYY-MM-DD`，用于排序和显示。
- **`draft`**：布尔值（`true` 或 `false`），用于标记页面是否为草稿。如果设置为 `true`，则在默认情况下不会被 Hugo 渲染到网站中。
- **`description`**：页面的简短描述，通常用于 SEO 和 Open Graph 标签。
- **`keywords`**：页面的关键字或标签，用于 SEO。
- **`author`**：页面的作者。
- **`categories`** 和 **`tags`**：用于对页面进行分类和标记，支持多值数组。
- **`series`**：将页面归类到某个系列中，通常用于多篇文章的集合。
- **`aliases`**：为页面设置别名或重定向路径。
- **`image`**：页面的封面图片，通常用于文章或页面的缩略图。

### **2. 页面布局和模板相关**
这些属性用于控制页面的布局和模板选择：

- **`layout`**：指定页面使用的布局模板。例如，`layout = "post"` 或 `layout = "page"`。
- **`type`**：指定页面的类型，例如 `post`、`page`、`section` 等。
- **`weight`**：用于控制页面在列表中的排序权重，值越小越靠前。
- **`menu`**：将页面添加到导航菜单中，支持自定义菜单名称、权重等。

### **3. SEO 和元数据**
这些属性用于优化页面的搜索引擎优化（SEO）和社交媒体分享：

- **`canonicalURL`**：设置页面的规范 URL。
- **`robots`**：控制页面是否被搜索引擎索引（例如 `index, follow` 或 `noindex, nofollow`）。
- **`ogImage`**：为社交媒体（如 Facebook 或 Twitter）设置 Open Graph 图片。
- **`summary`**：页面的摘要，用于 RSS 或页面列表显示。

### **4. 特定主题支持的属性**
许多 Hugo 主题会扩展 Front Matter 的功能，添加一些特定的属性。以下是一些常见的扩展属性：

- **`featuredImage`** 或 **`thumbnail`**：设置页面的特色图片。
- **`toc`**：布尔值，用于控制是否显示目录（Table of Contents）。
- **`comments`**：布尔值，用于控制是否启用评论功能。
- **`share`**：布尔值，用于控制是否显示分享按钮。
- **`related`**：布尔值，用于控制是否显示相关文章。
- **`math`**：布尔值，用于启用数学公式渲染（例如，通过 KaTeX 或 MathJax）。
- **`readingTime`**：布尔值，用于显示预计阅读时间。

### **5. 自定义属性**
Hugo 允许你在 Front Matter 中添加任意自定义属性，这些属性可以通过模板或短代码（Shortcodes）在页面中使用。例如：

```yaml
customField: "This is a custom field"

你可以在模板中通过 .Params.customField 访问它。

总结

Hugo 的 Front Matter 是一个强大的工具，可以让你灵活地控制页面的渲染和行为。通过合理使用这些属性，你可以优化网站的结构、SEO 和用户体验。如果你使用的是特定主题，建议查阅主题的文档，了解它支持的自定义 Front Matter 属性。
